00011 <h2>8. LIMIT</h2>
00012 <p>Sometimes you may not want all possible results.  The <em>LIMIT</em> clause allows you to limit how many results are returned.  In the examples used in <a href="?page=FILTER">FILTER</a> and <a href="?page=ORDER BY">ORDER BY</a> you may only want to the latest 10 workflows:</p>
00013 <div class="yellow"><pre>PREFIX dcterms: &lt;http://purl.org/dc/terms/&gt;
00014 PREFIX mecontrib: &lt;http://rdf.myexperiment.org/ontologies/contributions/&gt;
00015 PREFIX rdf: &lt;http://www.w3.org/1999/02/22-rdf-syntax-ns#&gt;
00016 SELECT ?workflow ?added
00017 WHERE { 
00018   ?workflow rdf:type mecontrib:Workflow ;
00019     dcterms:created ?added
00020 }
00021 ORDER BY DESC(?added)
00022 <b>LIMIT 10</b></pre>

00039 <h3><a name="OFFSET"/>8.1 OFFSET</h3>
00040 <p>Once you have got the first 10 workflows you might want to get the next 10. The <em>OFFSET</em> clause allows you to do this:</p>
00041 <div class="yellow"><pre>PREFIX dcterms: &lt;http://purl.org/dc/terms/&gt;
00042 PREFIX mecontrib: &lt;http://rdf.myexperiment.org/ontologies/contributions/&gt;
00043 PREFIX rdf: &lt;http://www.w3.org/1999/02/22-rdf-syntax-ns#&gt;
00044 SELECT ?workflow ?added
00045 WHERE {
00046   ?workflow rdf:type mecontrib:Workflow ;
00047     dcterms:created ?added
00048 }
00049 ORDER BY DESC(?added)
00050 LIMIT 10
00051 <b>OFFSET 10</b></pre>
