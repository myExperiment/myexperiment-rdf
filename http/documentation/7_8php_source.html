00011 <h2>7. ORDER BY</h2>
00012 <p>The <a href="?page=FILTER#On Dates">filter on dates example</a> restricts the workflows returned, however you may want to go further, listing the most recent workflows first.  The <em>ORDER BY</em> clause can be used to do this:</p>
00013 <div class="yellow"><pre>PREFIX xsd: &lt;http://www.w3.org/2001/XMLSchema#&gt;
00014 PREFIX dcterms: &lt;http://purl.org/dc/terms/&gt;
00015 PREFIX mecontrib: &lt;http://rdf.myexperiment.org/ontologies/contributions/&gt;
00016 PREFIX rdf: &lt;http://www.w3.org/1999/02/22-rdf-syntax-ns#&gt;
00017 SELECT ?workflow ?added
00018 WHERE { 
00019   ?workflow rdf:type mecontrib:Workflow ;
00020     dcterms:created ?added
00021   FILTER ( ?added &gt;= xsd:dateTime('2009-09-01T00:00:00Z') ) 
00022 }
00023 <b>ORDER BY DESC(?added)</b></pre>

00035 <p>The <em>DESC</em> operand means order the results in descending order, in this case, the latest workflows first.  If you wanted them ordered ascending you don't require any operand.  If you want a second criteria to order on, you just add this after the first.  An example of this might be order by workflow type with the latest of each type first:</p>
00036 <div class="yellow"><pre>PREFIX mebase: &lt;http://rdf.myexperiment.org/ontologies/base/&gt;
00037 PREFIX xsd: &lt;http://www.w3.org/2001/XMLSchema#&gt;
00038 PREFIX dcterms: &lt;http://purl.org/dc/terms/&gt;
00039 PREFIX mecontrib: &lt;http://rdf.myexperiment.org/ontologies/contributions/&gt;
00040 PREFIX rdf: &lt;http://www.w3.org/1999/02/22-rdf-syntax-ns#&gt;
00041 SELECT ?workflow ?added ?ct_title
00042 WHERE { 
00043   ?workflow rdf:type mecontrib:Workflow ;
00044     dcterms:created ?added ;
00045     mebase:has-content-type ?ct .
00046   ?ct dcterms:title ?ct_title 
00047   FILTER ( ?added &gt;= xsd:dateTime('2009-09-01T00:00:00Z') ) 
00048 }
00049 ORDER BY <b>?ct_title</b> DESC(?added)</pre>

00062 <p>By default, <em>ORDER BY</em> will treat the field(s) it is ordering on as alphanumeric.  In some cases it is necessary to treat a field as being numeric.  An example of this is a list of the most downloaded workflows.  Below is an example of how to treat <em>?downloaded</em> as a numeric field.</p>
00063 <div class="yellow"><pre>BASE &lt;http://www.myexperiment.org/&gt;
00064 PREFIX xsd: &lt;http://www.w3.org/2001/XMLSchema#&gt;
00065 PREFIX mevd: &lt;http://rdf.myexperiment.org/ontologies/viewings_downloads/&gt;
00066 PREFIX sioc: &lt;http://rdfs.org/sioc/ns#&gt;
00067 PREFIX rdf: &lt;http://www.w3.org/1999/02/22-rdf-syntax-ns#&gt;
00068 PREFIX mecontrib: &lt;http://rdf.myexperiment.org/ontologies/contributions/&gt;
00069 
00070 SELECT ?workflow ?downloaded
00071 WHERE {
00072   ?workflow rdf:type mecontrib:Workflow .
00073   ?workflow sioc:has_owner &lt;users/43&gt; .
00074   ?workflow mevd:downloaded ?downloaded
00075 }
